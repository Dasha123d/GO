# Введение в RPC в Go

RPC (Remote Procedure Call) — парадигма, позволяющая вызывать функции на удалённом сервере как локальные. В Go есть встроенный пакет `net/rpc`, а также популярный фреймворк gRPC.

## net/rpc (встроенный)

Стандартная библиотека поддерживает RPC через `encoding/gob`. Пример:

```go
type Args struct { A, B int }
type Arith int

func (t *Arith) Multiply(args *Args, reply *int) error {
    *reply = args.A * args.B
    return nil
}
rpc.Register(new(Arith))
```

Ограничения: только Go-клиенты, синхронные вызовы, отсутствует поддержка HTTP/2, потоков.

## gRPC
gRPC (Google) — современный кросс-языковой RPC-фреймворк на основе Protocol Buffers и HTTP/2. Поддерживает:
* Строгую типизацию через .proto-контракты.
* Четыре режима взаимодействия: унарный, серверный стриминг, клиентский стриминг, двунаправленный стриминг.
* Автоматическую генерацию клиентского и серверного кода.
* Аутентификацию, перехватчики, балансировку нагрузки.

В этом разделе мы сосредоточимся на gRPC, так как он является индустриальным стандартом для микросервисного взаимодействия в Go.
