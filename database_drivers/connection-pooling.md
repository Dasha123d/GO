# Пул соединений и управление временем жизни

Объект `*sql.DB` представляет собой пул открытых соединений с базой данных. Он безопасен для параллельного использования и автоматически открывает, переиспользует и закрывает соединения.

## Основные параметры пула

```go
db, err := sql.Open("postgres", dsn)
if err != nil {
    log.Fatal(err)
}

// Максимальное количество открытых соединений (по умолчанию не ограничено, но лучше задать)
db.SetMaxOpenConns(25)

// Максимальное количество простаивающих соединений в пуле (по умолчанию 2)
db.SetMaxIdleConns(10)

// Максимальное время жизни соединения с момента создания (по умолчанию бесконечно)
db.SetConnMaxLifetime(5 * time.Minute)

// Максимальное время простоя соединения (Go >= 1.15)
db.SetConnMaxIdleTime(1 * time.Minute)```

* SetMaxOpenConns ограничивает одновременное количество физических соединений к БД. Превышение приведёт к блокировке вызовов Exec/Query до освобождения соединения. Рекомендуется ставить значение чуть выше ожидаемой пиковой нагрузки.
* SetMaxIdleConns – сколько соединений держать открытыми в режиме ожидания. Слишком маленькое число приведёт к частому открытию/закрытию, слишком большое – расходует память.
* SetConnMaxLifetime – гарантирует, что драйвер не будет использовать соединение дольше указанного времени (полезно для ротации сертификатов, балансировки). По истечении срока соединение закрывается и заменяется новым.
*SetConnMaxIdleTime – если соединение простаивает дольше этого времени, оно закрывается. Позволяет снизить количество idle-соединений при спаде нагрузки.

## Как пул работает
1. При вызове db.Query() / db.Exec() запрашивается соединение из пула.
2. Если есть свободное соединение (idle), оно используется.
3. Если свободных нет, но текущее количество открытых соединений < MaxOpenConns, открывается новое.
4. Иначе горутина блокируется в ожидании освобождения соединения (с учётом контекста).
5. После завершения работы с соединением оно возвращается в пул (становится idle) или закрывается, если его время жизни истекло или превышен лимит idle.

## Мониторинг
Пакет предоставляет статистику через db.Stats():
```go
stats := db.Stats()
fmt.Printf("Open: %d, InUse: %d, Idle: %d, WaitCount: %d, WaitDuration: %v\n",
    stats.OpenConnections, stats.InUse, stats.Idle,
    stats.WaitCount, stats.WaitDuration)```

* OpenConnections – текущее число открытых соединений.
* InUse – сколько из них активно.
* Idle – сколько простаивает.
* WaitCount / WaitDuration – сколько раз и как долго ожидали соединения.

Можно периодически логировать для выявления узких мест.

## Контекст и таймауты
Всегда используйте варианты методов с контекстом: QueryContext, ExecContext, BeginTx. Это позволяет задать таймаут на операцию и корректно отменять долгие запросы.
```go
ctx, cancel := context.WithTimeout(context.Background(), 2*time.Second)
defer cancel()
rows, err := db.QueryContext(ctx, "SELECT ...")```

## Лучшие практики
* Устанавливайте SetMaxOpenConns в зависимости от возможностей БД (например, max_connections PostgreSQL минус запас).
* Не создавайте новый *sql.DB для каждого запроса – объект должен быть долгоживущим (один на приложение).
* Всегда закрывайте db.Close() при graceful shutdown, чтобы не рвать соединения.
* Проверяйте db.Ping() при старте приложения.
* Используйте метрики из db.Stats() для мониторинга состояния пула.
