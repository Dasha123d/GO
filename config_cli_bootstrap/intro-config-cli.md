# Введение в конфигурацию и CLI в Go

Приложение на Go обычно требует внешней конфигурации (параметры подключения, порты, токены) и часто запускается из командной строки с различными опциями. Разработчики применяют библиотеки для упрощения работы: `viper` для конфигурации, `cobra` для CLI.

## Почему это важно

- **Отделение кода от настроек** упрощает развёртывание.
- **Поддержка разных окружений** (development, staging, production) через разные файлы или переменные.
- **Удобный CLI** помогает автоматизации и DevOps.

## Типичный стек

- *Конфигурация*: `spf13/viper` (файлы + env + флаги), `kelseyhightower/envconfig` (только env), встроенные `flag` и `os.Getenv`.
- *CLI*: `spf13/cobra`, реже — стандартный `flag`, `urfave/cli`.
- *Bootstrap*: структурирование `main.go`, инициализация зависимостей, graceful shutdown.
