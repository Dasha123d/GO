# Введение в WebSocket в Go

WebSocket — протокол полнодуплексной связи поверх TCP, начинающийся с HTTP-апгрейда. В Go нет встроенной полной реализации WebSocket в стандартной библиотеке, но есть примитивы для апгрейда через `net/http` и популярные библиотеки: `gorilla/websocket` и `nhooyr.io/websocket`.

## Как работает WebSocket

1. Клиент отправляет HTTP-запрос с заголовками `Upgrade: websocket` и `Connection: Upgrade`.
2. Сервер отвечает `101 Switching Protocols`, после чего соединение переходит в режим WebSocket.
3. Данные передаются кадрами (text, binary, ping/pong, close).
4. Закрытие инициируется специальным кадром.

## Стандартная библиотека: `net/http`

HTTP-сервер Go позволяет захватить базовое соединение через `http.Hijacker`, но вручную реализовывать фрейминг сложно и не рекомендуется. Вместо этого используют готовые библиотеки.

## Выбор библиотеки

- **gorilla/websocket** — де-факто стандарт, богатый API, поддержка подпротоколов, сжатия, проверка origin. Хорошо подходит для чатов, игр, real-time уведомлений.
- **nhooyr.io/websocket** — современная, минималистичная, активно развиваемая библиотека с акцентом на контексты и безопасность. Меньше зависимостей.
